NgRx Reducers

Definition

Reducers are pure functions that take the current state and an action as arguments, and return a new state.

They specify how the application's state changes in response to actions sent to the store. Reducers must be pure functions with no side effects.

Each reducer is responsible for managing its own part of the global state tree.

Code Examples

Reducer Implementation
State Interface
// State interface
export interface ItemState {{ '{' }}
  items: Item[];
  loading: boolean;
  error: string | null;
}

export const initialState: ItemState = {{ '{' }}
  items: [],
  loading: false,
  error: null
};
Reducer Function
export const itemReducer = createReducer(
  initialState,
  on(addItem, (state, {{ '{' }} item }) => ({{ '{' }}
    ...state,
    items: [...state.items, item]
  })),
  on(removeItem, (state, {{ '{' }} id }) => ({{ '{' }}
    ...state,
    items: state.items.filter(item => item.id !== id)
  }))
);
Key Principles
Pure

No side effects, same input always produces same output

Immutable

Never mutate state directly, always return new state object

Predictable

State changes are predictable and traceable

Testable

Easy to unit test due to pure function nature
