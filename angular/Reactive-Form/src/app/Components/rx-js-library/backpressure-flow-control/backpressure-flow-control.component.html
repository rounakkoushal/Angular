Backpressure & Flow Control

Definition

Backpressure occurs when data is produced faster than it can be consumed, leading to memory issues and performance problems.

Flow control operators in RxJS help manage the rate of data emission and handle scenarios where producers and consumers operate at different speeds.

Key operators include throttle, debounce, sample, buffer, and window for controlling data flow and preventing system overload.

Flow Control Operators
  • throttle/throttleTime - Rate limiting
  • debounce/debounceTime - Delay emissions
  • sample/sampleTime - Periodic sampling
  • buffer/bufferTime - Collect emissions
Common Scenarios
  • Search input optimization
  • Button click prevention
  • Scroll event handling
  • Real-time data streams
  • API rate limiting

Code Examples

Implementation Examples
Throttle Example
// Throttle scroll events
fromEvent(window, 'scroll').pipe(
  throttleTime(100), // Max once per 100ms
  map(() => window.scrollY)
).subscribe(scrollY => {
  console.log('Scroll position:', scrollY);
});
Debounce Example
// Debounce search input
fromEvent(searchInput, 'input').pipe(
  debounceTime(300), // Wait 300ms after last input
  map(event => event.target.value),
  distinctUntilChanged()
).subscribe(searchTerm => {
  performSearch(searchTerm);
});
Buffer Example
// Buffer clicks for batch processing
fromEvent(button, 'click').pipe(
  bufferTime(1000), // Collect clicks for 1 second
  filter(clicks => clicks.length > 0)
).subscribe(clickBatch => {
  console.log(`${clickBatch.length} clicks`);
});
Sample Example
// Sample mouse position periodically
fromEvent(document, 'mousemove').pipe(
  sampleTime(100), // Sample every 100ms
  map(event => ({ x: event.clientX, y: event.clientY }))
).subscribe(position => {
  updateCursor(position);
});
