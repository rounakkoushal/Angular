Higher-order Observables

Definition

Higher order observables is a observable that emits other observables.

High-order mapping operators transform higher-order Observables. They map each value from an outer Observable to a new inner Observable and automatically subscribe to, and unsubscribe from that inner Observable.

RxJS provides several higher-order mapping operators. You can recognize a higher-order mapping operator by its name, which ends in map, such as mergeMap, switchMap, concatMap, exhaustMap etc.

Common Use Cases
  • HTTP requests triggered by user input
  • Nested API calls with dependencies
  • Dynamic data streams
  • Real-time search functionality
  • File upload with progress tracking
Flattening Operators
mergeMap Merge all inner observables concurrently
switchMap Switch to latest inner observable
concatMap Concatenate inner observables sequentially
exhaustMap Ignore new while current is active

Code Examples

Implementation Examples
MergeMap Example
// Concurrent processing
const clicks$ = fromEvent(button, 'click');
clicks$.pipe(
  mergeMap(() => 
    interval(1000).pipe(take(3))
  )
).subscribe(console.log);
SwitchMap Example
// Cancel previous, use latest
searchInput$.pipe(
  switchMap(query => 
    this.http.get(`/search?q=${{ '{' }}query}`)
  )
).subscribe(results => 
  console.log(results)
);
ConcatMap Example
// Sequential processing
requests$.pipe(
  concatMap(request => 
    this.http.post('/api', request)
  )
).subscribe(response => 
  console.log(response)
);
ExhaustMap Example
// Ignore while busy
loginClicks$.pipe(
  exhaustMap(() => 
    this.auth.login(credentials)
  )
).subscribe(result => 
  console.log(result)
);
Key Takeaways
mergeMap

Use when order doesn't matter and you want concurrent execution

switchMap

Use for search, autocomplete, or when only latest result matters

concatMap

Use when order matters and you need sequential execution

exhaustMap

Use to prevent duplicate operations like login attempts
