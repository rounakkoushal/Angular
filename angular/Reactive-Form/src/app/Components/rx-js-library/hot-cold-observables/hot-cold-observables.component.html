Hot vs Cold Observables

Definition

Hot and Cold Observables represent different execution models in RxJS.

Cold Observables are unicast and lazy - each subscriber gets its own independent execution. Hot Observables are multicast and eager - all subscribers share the same execution.

Understanding this difference is crucial for managing subscriptions, preventing memory leaks, and optimizing performance in reactive applications.

Cold Observables
  • Unicast (one-to-one)
  • Lazy execution
  • Each subscriber gets its own execution
  • Data producer is created inside Observable
Examples: HTTP requests, timer(), interval(), of(), from()
Hot Observables
  • Multicast (one-to-many)
  • Eager execution
  • All subscribers share the same execution
  • Data producer exists outside Observable
Examples: DOM events, WebSocket connections, Subject, BehaviorSubject

Code Examples

Implementation Examples
Cold Observable Example
// Each subscriber gets independent execution
const coldObservable = new Observable(subscriber => {
  console.log('Observable execution started');
  const data = Math.random();
  subscriber.next(data);
  subscriber.complete();
});

// Different values for each subscriber
coldObservable.subscribe(console.log);
Hot Observable Example
// Shared execution using Subject
const subject = new Subject();
const hotObservable = subject.asObservable();

// Subscribe before emitting
hotObservable.subscribe(console.log);

// Emit values - all subscribers get same values
subject.next('Hello World');
Converting Cold to Hot
// Using share() operator
const coldTimer = timer(0, 1000);
const hotTimer = coldTimer.pipe(share());

// Multiple subscribers share same timer
hotTimer.subscribe(console.log);
Using shareReplay()
// Replay last N values to new subscribers
const replayed = source$.pipe(shareReplay(1));
replayed.subscribe(console.log);
// Late subscribers get last value immediately
