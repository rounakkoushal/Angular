Subjects & Multicasting

Definition

Subjects are special types of Observables that allow values to be multicasted to many Observers.

Unlike regular Observables, Subjects are both Observable and Observer - they can emit values and be subscribed to. This makes them perfect for multicasting scenarios where multiple subscribers need to receive the same values.

RxJS provides several Subject types: Subject, BehaviorSubject, ReplaySubject, and AsyncSubject, each with unique characteristics for different use cases.

Subject Types
  • Subject - Basic multicasting
  • BehaviorSubject - Stores current value
  • ReplaySubject - Replays N values
  • AsyncSubject - Emits last value on complete
Key Characteristics
  • Hot Observables by nature
  • Multicast to multiple subscribers
  • Can emit values imperatively
  • Act as both Observable and Observer
  • Perfect for event buses and state management

Code Examples

Implementation Examples
Subject Example
// Basic Subject
const subject = new Subject<number>();

// Subscribe before emitting
subject.subscribe(value => console.log('Sub 1:', value));
subject.subscribe(value => console.log('Sub 2:', value));

// Emit values
subject.next(1);
subject.next(2);
subject.complete();
BehaviorSubject Example
// BehaviorSubject with initial value
const behavior = new BehaviorSubject<string>('initial');

// Late subscriber gets current value
behavior.subscribe(console.log); // 'initial'

behavior.next('updated');
// New subscriber gets latest value
behavior.subscribe(console.log); // 'updated'
ReplaySubject Example
// ReplaySubject with buffer size 2
const replay = new ReplaySubject<number>(2);

replay.next(1);
replay.next(2);
replay.next(3);

// Late subscriber gets last 2 values
replay.subscribe(console.log); // 2, 3
AsyncSubject Example
// AsyncSubject emits only last value
const async = new AsyncSubject<number>();

async.subscribe(console.log);
async.next(1);
async.next(2);
async.next(3);
async.complete(); // Only emits 3
Common Use Cases
Subject

Event buses, notifications, simple multicasting

BehaviorSubject

State management, current user, configuration settings

ReplaySubject

Caching, chat history, recent activity logs

AsyncSubject

Final results, completion status, last computed value
