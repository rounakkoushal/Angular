Testing & Marble Diagrams

Definition

Marble diagrams are visual representations of Observable streams over time, showing when values are emitted, errors occur, or streams complete.

RxJS TestScheduler enables marble testing, allowing you to test complex asynchronous Observable behavior synchronously with virtual time control.

Marble syntax uses characters to represent emissions: letters/numbers for values, | for completion, # for errors, and - for time frames.

Marble Syntax
  • a, b, c - Emitted values
  • - - Time frame (10ms)
  • | - Completion
  • # - Error
  • ^ - Subscription point
Testing Benefits
  • Synchronous async testing
  • Virtual time control
  • Precise timing verification
  • Complex scenario testing
  • Visual test documentation

Code Examples

Testing Examples
Basic Marble Test
testScheduler.run(({ cold, expectObservable }) => {
  const source$ = cold('a-b-c|');
  const expected = 'a-b-c|';
  
  expectObservable(source$).toBe(expected);
});
Operator Testing
testScheduler.run(({ cold, expectObservable }) => {
  const source$ = cold('a-b-c|');
  const result$ = source$.pipe(map(x => x.toUpperCase()));
  const expected = 'A-B-C|';
  
  expectObservable(result$).toBe(expected);
});
Error Testing
testScheduler.run(({ cold, expectObservable }) => {
  const source$ = cold('a-b-#');
  const expected = 'a-b-#';
  
  expectObservable(source$).toBe(expected);
});
Subscription Testing
testScheduler.run(({ cold, expectSubscriptions }) => {
  const source$ = cold('a-b-c|');
  const subs = '^----!';
  
  expectSubscriptions(source$.subscriptions).toBe(subs);
});
Common Marble Patterns
a-b-c|

Simple emission sequence with completion

a---b---c

Delayed emissions with gaps

a-b-#

Emission sequence ending with error

^--!

Subscription and unsubscription timing
